# Publication Workflow

This repository is the public priority package for the Golden Horizon Principle.

The full local working master remains private unless Peter explicitly requests a specific file be made public.

## Sync Rule

When the local GHP files are updated on Peter's computer, the public package should also be checked.

If the update is public-facing, this repo should receive a distilled, status-labeled version of the change.

If the update is private, internal, raw, strategic, or part of the full working organism, it should remain local.

## Public Package Rule

Public updates should preserve:

- claim status,
- do-not-claim guardrails,
- toy-telemetry limits,
- symbolic-vs-scientific separation,
- and the distinction between the public priority package and the private master.

## Never Publish By Default

Do not publish by default:

- the full `GHP_v1_618_MASTER.md`,
- raw AI chats,
- messy addenda,
- private strategy,
- Holographic Resonator code,
- credentials,
- or private working files.

## Public Release Discipline

When public-facing files change:

1. update local public package files,
2. commit the public package,
3. push to GitHub,
4. tag or release meaningful checkpoints,
5. report what changed and what stayed private.
